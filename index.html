Git Learning Guide

From Beginner to Professional Developer

Phase 1: Basics Phase 2: Essential Skills Phase 3: Intermediate Phase 4: Advanced

Phase 1: Git Basics

Lesson 1: What is Git?

Description: Git is a distributed version control system that tracks changes to files and allows collaboration among developers. 

git --version

Exercise: Write a 2-sentence explanation of Git in your own words.

Lesson 2: Installing Git

Description: Set up Git on your system and configure user settings. 

sudo apt install git
brew install git
git config --global user.name "Your Name"
git config --global user.email "you@example.com"

Exercise: Install Git and set your global username and email.

Lesson 3: Basic Git Workflow

Description: Learn how to create a repository, stage changes, and make commits. 

git init
git add .
git commit -m "Initial commit"
git status
git log

Exercise: Create a repository, add a file, and make your first commit.

Lesson 4: Understanding Commits

Description: Commits are snapshots of your project history, identified by unique hashes. 

git commit -m "Add feature A"
git log

Exercise: Make three commits and view them with git log.

Lesson 5: Branching Basics

Description: Branches allow isolated development without affecting the main codebase. 

git branch feature-1
git checkout feature-1

Exercise: Create two branches and switch between them using git checkout.

Phase 2: Essential Git Skills

Lesson 6: Cloning Repositories

Description: Clone existing repositories to work locally. 

git clone https://github.com/user/repository.git

Exercise: Clone a public GitHub repository to your machine.

Lesson 7: Pull and Fetch

Description: Understand the difference between pulling changes and fetching updates from a remote. 

git fetch
    git pull

Exercise: Fetch and pull changes from a remote repository after you clone it.

Lesson 8: Pushing Changes

Description: Push your commits from the local repository to the remote server. 

git push origin main

Exercise: Push a new commit to a GitHub repository.

Lesson 9: Merging Branches

Description: Merge a branch into another to combine development work. 

git checkout main
git merge feature-branch

Exercise: Merge a feature branch into your main branch.

Lesson 10: Resolving Merge Conflicts

Description: Handle conflicts that occur when merging two branches. 

git status
git add conflicted-file
git commit

Exercise: Create a conflict between two branches and resolve it manually.

Phase 3: Intermediate Git

Lesson 11: Stashing Changes

Description: Temporarily store changes without committing them. 

git stash
git stash apply

Exercise: Make a change, stash it, and reapply it later.

Lesson 12: Working with Remotes

Description: Add and manage remote repositories. 

git remote add origin https://github.com/user/repo.git
git remote -v

Exercise: Add a new remote repository to a local project.

Lesson 13: Reverting and Resetting

Description: Undo changes using revert and reset commands. 

git revert commit_id
git reset --hard commit_id

Exercise: Revert a commit, then try a hard reset to a previous state.

Lesson 14: Tagging Releases

Description: Use tags to mark important points in history like releases. 

git tag v1.0
git push origin v1.0

Exercise: Create a tag and push it to GitHub.

Lesson 15: Cherry Picking

Description: Apply commits from one branch onto another. 

git cherry-pick commit_id

Exercise: Cherry-pick a commit from one branch into another branch.

Phase 4: Advanced Git

Lesson 16: Git Rebase

Description: Move or combine a sequence of commits to a new base commit. 

git checkout feature-branch
git rebase main

Exercise: Rebase a feature branch onto main.

Lesson 17: Squash Commits

Description: Combine multiple commits into a single commit during a rebase. 

git rebase -i HEAD~3

Exercise: Squash your last three commits into one.

Lesson 18: Git Hooks

Description: Scripts that run automatically at specific Git events. 

cd .git/hooks
ls

Exercise: Explore existing Git hooks and set up a simple pre-commit hook.

Lesson 19: Submodules

Description: Include other Git repositories inside a main repository. 

git submodule add https://github.com/example/repo.git

Exercise: Add a submodule to your project.

Lesson 20: Advanced Troubleshooting

Description: Diagnose and fix Git issues using advanced commands. 

git fsck
git reflog

Exercise: Use reflog to recover a deleted branch or lost commit.
